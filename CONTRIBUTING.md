# Contributing

Thanks for wanting to pitch in. Two ground rules before your first PR — they exist because of how
this project is licensed (source-available, with commercial rights reserved; see [LICENSE](LICENSE)
and the README's License section).

By submitting a contribution to this repository (a pull request, patch, or code suggestion) you
agree that:

1. **Inbound = outbound.** Your contribution is licensed to this project and to everyone who
   receives it under the same [PolyForm Noncommercial License 1.0.0](LICENSE) that covers the rest
   of the code.
2. **Commercial-rights grant.** You additionally grant Jordan Curran a perpetual, irrevocable,
   worldwide, royalty-free license to use, reproduce, modify, distribute, sublicense, relicense,
   and commercially exploit your contribution as part of this project. You keep the copyright to
   your work; this is a license, not an assignment.

Rule 2 is the unusual one, so here's the honest why: the noncommercial license only works as a
"self-host free, commercial rights stay with the author" model if one person can license the
*whole* work commercially. Without this grant, the first outside patch would fragment that — a
commercial license couldn't cover your lines without tracking you down for a signature later. This
is the standard trade-off of contributing to a source-available project; if it's not for you,
noncommercial forks are exactly what the license is for.

Practical notes: open an issue first for anything bigger than a small fix, and the bar for code is
the one the repo already shows — tests with behavior changes, no swallowed exceptions, no dead
code ([CLAUDE.md](CLAUDE.md) has the details).
